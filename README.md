# Цель проекта

Разработать приложение для анализа транзакций, Excel-файла. 
Приложение генерирует JSON-данные для веб-страниц, формирует Excel-отчеты, 
а также предоставлять другие сервисы.

# инструкция по установке и использованию разработанных функций

1. Клонируйте репозиторий:
```
git clone git@github.com:4usnok/4usnok-Skypro_python_prj1.git
```
2. Установите зависимости:
```
pip install -r requirements.txt
```

3. Список разработанных модулей:
* Модули для реализации функционала проекта, находятся в директории src
* Модуль views:
отвечает за данные для анализа и вывода на веб-страницах — это данные с начала месяца, на который выпадает 
входящая дата, по входящую дату. Если дата — 20.05.2020, то данные для анализа будут в диапазоне 
01.05.2020-20.05.2020. Валюты и акции для отображения на веб-страницах задаются в отдельном файле 
пользовательских настроек user_settings.json, который находится в корне проекта.
* Модуль services:
отвечает за сервисы, который позволяет проанализировать, 
какие категории были наиболее выгодными для выбора в качестве категорий повышенного кешбэка.
* Модуль reports:
в модуле содержится декоратор, который записывает в файл результат, 
который возвращается функция, формирующая отчет.
* Модуль utils:
содержит в себе функции, которые облегчают работу с модулями.
* Модуль main.py:
является управляющим и содержит в себе вызовы функций модулей: reports, services, views

# Принцип работы функций в модуле views.py
* greetings - функция получает на вход время в формате YYYY-MM-DD HH:MM:SS и выводит сообщение:
«Доброе утро» / «Добрый день» / «Добрый вечер» / «Доброй ночи» в зависимости от текущего времени.
* for_each_card - функция получает на вход время в формате YYYY-MM-DD HH:MM:SS и выводит:
последние 4 цифры карты, общую сумму расходов и кешбэк (1 рубль на каждые 100 рублей). Все данные берутся из файла 
operations.xlsx, который находится в папке data в корне проекта
* top_trans - функция получает на вход время в формате YYYY-MM-DD HH:MM:SS и 
выводит топ-5 транзакций по сумме платежа в диапазоне от начала месяца до заданной даты.
* cur_proc и stock_processing - функции получают на вход время в формате YYYY-MM-DD HH:MM:SS, после чего
делают реквест-запрос к своим api и выводят данные о курсе валют и акциях. Данные для api берутся из 
user_settings.json файла с настройками, который находится в корне проекта
* main_views - является главной и управляющей функцией, которая генерирует данные из функций модуля в json ответ

# Принцип работы функций в модуле services.py:
* filter_transactions_by_date - функция отвечает за фильтрацию транзакции по месяцу и году. Является вспомогательной.
* sum_cashback_by_category - функция суммирует кешбэк по категориям. Является вспомогательной
* favorable_categories_of_increased_cashback - функция сервиса «Выгодные категории повышенного кешбэка» принимает год, 
месяц и транзакции в формате списка словарей. Отдаёт json ответ, который помогает определить наиболее выгодные для выбора
в качестве категорий повышенного кешбэка

в определённой категории за последние три месяца.
* Модуль содержит в себе декоратор, который помогает улучшить функционал:
export_to_file - Декоратор для экспорта результата функции в файл. Поддерживает JSON, CSV и Excel.

# Инструкция по тестированию программы на примере 
1. Зайдите в пакет tests
2. На выбор представлено 3 модуля, которые тестируют функционал модулей и их функций
3. Каждая функция, помимо своего параметра, также имеет специальную инструкцию assert, которая принимает на вход 
выражение, значением которого должно быть True, иначе выбрасывается исключение.
4. Как только ввели свои данные, введите в консоли ключевое слово pytest
5. В результате, сможем увидеть: какое количество функций работает корректно.
6. Функции тестируются с помощью фреймворков pytest, unittest
